# log16 user levels v01

## Краткий смысл

Один публичный repo должен обслуживать разные типы людей. Нельзя писать его так, будто все уже умеют жить в bash и дружат с локальными моделями.

## Level 0. Reader

Кто:

    просто читает
    не запускает код
    хочет понять проект

Что нужно:

    short README
    project overview
    FAQ
    examples of answers
    glossary

## Level 1. GitHub user

Кто:

    умеет читать repo
    умеет issues
    может открыть docs
    не готов ставить локальный AI

Что нужно:

    docs index
    tasks list
    contribution map
    no-install demo explanation
    public knowledge base

## Level 2. Local AI tester

Кто:

    готов поставить Ollama
    готов скачать qwen3:8b
    готов запускать scripts
    может прислать logs

Что нужно:

    local-ai onboarding
    environment doctor
    minimal install
    mock/demo mode
    common errors

## Level 3. Developer

Кто:

    читает Python/bash
    может писать tests
    может делать PR
    может разбирать schemas

Что нужно:

    architecture docs
    module map
    tests
    coding conventions
    issue templates
    contribution rules

## Level 4. Infrastructure operator

Кто:

    умеет deploy
    понимает hosting
    может поднять service
    понимает security/cost

Что нужно:

    hosted architecture
    deployment plan
    resource estimate
    security boundaries
    rate limits
    backup policy

## Level 5. Project participant

Кто:

    интересуется участием в БЛАГОПОЛУЧИИ
    хочет понять, куда приложить работу
    не обязательно программист

Что нужно:

    participant pathway
    task classes
    role map
    minimal useful actions
    communication rules

## Вывод

Публичный repo должен быть не одной дверью, а лестницей.
