# Epistemic gate v01: слой проверки состояния знания

## Краткий смысл

`epistemic gate` — это первый слой, который должен срабатывать до прямого ответа или генерации задач.

Его задача: определить, может ли система дать проверяемый ответ, или сначала нужно пополнить файловое поле, запросить Сущность, собрать источники или получить решение ОПЕРАТОРА.

## Почему это нужно

Без такого слоя широкий вопрос попадает либо в `answer`, где получает сухой отказ `unsupported question type`, либо сразу в `tasks`, где система начинает порождать работу под один конкретный запрос.

Это неправильно. Сначала система должна честно определить состояние знания:

- знаю и могу ответить;
- знаю частично;
- не знаю, потому что не хватает источников;
- нужна информация в файловом поле;
- нужен ответ Сущности;
- нужно решение ОПЕРАТОРА.

## Статусы v01

- `KNOWN_ANSWER` — ответ можно дать по имеющимся источникам;
- `PARTIAL_ANSWER` — можно дать предварительный ответ, но есть пробелы;
- `INSUFFICIENT_EVIDENCE` — проверяемого ответа нет, нужны источники;
- `NEEDS_FILE_FIELD` — нужная информация должна быть внесена в файловое поле;
- `NEEDS_ENTITY_RESPONSE` — нужен ответ профильной Сущности;
- `NEEDS_OPERATOR_DECISION` — нужен выбор или утверждение ОПЕРАТОРА;
- `UNSUPPORTED_MODE` — вопрос технически не поддержан текущей версией.

## Пример

Команда:

```bash
log16-epistemic-gate "Как произвести инвентаризацию файлового поля проекта?"
```

Ожидаемый смысл ответа:

```text
Я не могу дать проверяемый ответ: в файловом поле недостаточно подтверждённой информации.

Статус: NEEDS_FILE_FIELD

Нужны:
- карта корневых каталогов;
- правила классификации файлов;
- критерии завершённости инвентаризации;
- карта ответственности Сущностей.
```

## Ограничения v01

v01 пока не выполняет полноценный поиск по источникам. Это консервативный классификатор состояния знания и маршрута дальнейшей работы.
