# Publication boundary

## Краткий смысл

Этот документ задаёт границу между публичной базой и внутренним рабочим контуром проекта.

## Можно публиковать

    объяснение проекта
    публичные FAQ
    общие маршруты участия
    классы задач
    публичный статус работ
    glossary
    approved/reusable answers

## Нельзя публиковать без отдельной проверки

    персональные данные
    внутренние runtime queues
    служебные журналы
    рабочие outbox/inbox пакеты
    приватные адреса доступа
    raw entity outputs
    черновики без review
    внутренние конфликты
    неутверждённые внешние обещания

## Правило статуса

Публичная база должна различать:

    draft
    reviewed
    approved
    outdated
    needs_revision

## Правило честности

Если функция не готова, она должна быть описана как неготовая.

Если контур работает только локально, нельзя описывать его как доступный всем.
