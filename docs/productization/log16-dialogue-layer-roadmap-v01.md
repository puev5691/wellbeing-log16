# Диалоговый слой log16: ближайшее направление разработки

## Краткий смысл

Текущий `wellbeing-log16` уже имеет отдельные рабочие режимы: `answer`, `tasks`, status/dashboard и review. Но эти режимы пока требуют от ОПЕРАТОРА ручного понимания, куда именно отправлять вопрос.

Следующий этап — создать диалоговый слой, который будет принимать обычный человеческий запрос и сам выбирать нужный режим работы.

## Проблема текущей версии

Сейчас широкий вопрос может попасть в `answer` и получить честный, но малополезный отказ: `unsupported question type`.

Это технически правильно, потому что прямой ответ без источников и проверки был бы выдумкой. Но для ОПЕРАТОРА это неудобно: система должна не просто отказывать, а объяснять маршрут дальнейшей работы.

## Цель dialogue layer

Диалоговый слой должен:

1. принять обычный текстовый запрос;
2. определить тип намерения;
3. выбрать подходящий режим;
4. запустить нужный backend;
5. вернуть человеку понятный результат;
6. при нехватке данных создать задачу, запрос к Сущности или review-point.

## Первые классы намерений

- `direct_answer` — простой локальный ответ;
- `topic_to_task` — большая тема, которую нужно превратить в задачи;
- `status_lookup` — запрос состояния;
- `evidence_lookup` — нужен поиск источников и подтверждений;
- `review_request` — проверка результата;
- `entity_request` — вопрос или задание конкретной Сущности;
- `unsupported_with_route` — прямой ответ невозможен, но можно объяснить следующий рабочий шаг.

## Минимальный приемлемый результат v01

Команда вида:

```bash
log16-dialogue "Как произвести инвентаризацию файлового поля проекта?"
```

должна не возвращать сухой отказ, а отвечать примерно так:

```text
Этот вопрос требует не direct answer, а topic-to-task pipeline.
Созданы/предложены задачи:
1. определить границы файлового поля;
2. собрать список корневых каталогов;
3. построить классификацию типов файлов;
4. выделить мусор, дубли, runtime state и ценные документы;
5. передать задачи Архивариусу и КООРДИНАТОРУ.
```

## Статус

Рабочая постановка. Не реализовано в полной мере.
